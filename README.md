# Лабораторная работа №6
## Непрерывная интеграция и упаковка проектов с помощью CPack

**Студент:** Сардарян Гор ИУ8-22

## Цель работы
Настроить систему упаковки CPack для генерации установочных пакетов различных форматов (DEB, RPM, TGZ, ZIP) и автоматизировать создание релизов на GitHub с использованием GitHub Actions при пуше тегов.

## Выполненные шаги
1. Создан публичный репозиторий `lab06`. В него скопированы все необходимые библиотеки (`formatter_lib`, `formatter_ex_lib`, `solver_lib`, `solver_application`) из предыдущих лабораторных работ.
2. Доработан корневой файл `CMakeLists.txt`:
   - Добавлена установка исполняемого файла `solver` в `bin`.
   - Введена переменная `SOLVER_VERSION` (значение берётся из `version.txt` или задаётся вручную).
3. Создан файл `CPackConfig.cmake`, в котором указаны:
   - контактная информация и имя пакета;
   - генераторы пакетов (`DEB`, `RPM`, `TGZ`, `ZIP`);
   - специфичные для Debian и RPM настройки (maintainer, архитектура и т.д.);
   - список директорий, включаемых в исходные архивы.
4. Настроен GitHub Actions workflow (файл `.github/workflows/ci.yml`):
   - Триггер: `push` на теги, начинающиеся с `v*`.
   - Установка необходимых инструментов: `rpm-build` (для создания RPM-пакетов).
   - Конфигурация CMake с параметром `-DCPACK_GENERATOR="DEB;RPM;TGZ;ZIP"`.
   - Сборка пакетов командой `cpack`.
   - Создание релиза на GitHub с помощью действия `softprops/action-gh-release` и прикрепление к нему всех сгенерированных пакетов.
5. Исправлены ошибки:
   - В `CMakeLists.txt` добавлена явная зависимость `formatter_ex_lib` от `formatter_lib` (чтобы избежать проблем с линковкой).
   - Для Debian-пакета указан обязательный параметр `CPACK_DEBIAN_PACKAGE_MAINTAINER`.
6. Созданы теги `v1.0.0`, `v1.0.1`, `v1.0.2`, `v1.0.3`, `v1.0.4`, `v1.0.5`. Последний тег (`v1.0.5`) успешно запустил workflow и создал релиз.
